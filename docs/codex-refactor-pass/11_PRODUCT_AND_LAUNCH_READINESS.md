# Product And Launch Readiness

## Classification

Bounty Hunter is currently private/experimental but portfolio-worthy with cleanup. It is not public-launchable or paid-user ready yet.

## Strongest Product Value

The app has a clear loop for close relationships: missions create accountability, proof creates trust, approval creates recognition, and credits make rewards meaningful.

## Weakest Product Area

Trust boundaries. Credits, task lifecycle, storage, and notifications need server/database enforcement before public or monetized use.

## Simplest Monetizable Version

A private family/couple/friend-group app with:

- One small group.
- Missions and proof.
- Internal credits only.
- Custom rewards.
- No public marketplace.
- Paid upgrade later for multiple groups, themes, media storage, and richer reward templates.

## Blocks Launch

- Production migration state unknown.
- Storage bucket/policy setup incomplete.
- Task direct updates remain too broad.
- Legacy email functions are under-authorized.
- No automated tests for credits/rewards/proof.
- Build warnings remain for unresolved asset, bundle size, and mixed reward-domain import.

## Blocks iOS/PWA Conversion

- Mobile proof/upload/manual QA incomplete.
- Bundle size warning.
- Auth redirect URLs in Supabase config still reference old local ports.
- No documented app-store/privacy policy posture.
- Public/private storage decision unresolved for proof media.

## Blocks Paid Users

- No payment provider.
- No tenant/group model.
- No audit/admin/support tooling.
- No abuse/rate limiting.
- No reliable production migration/runbook proof.

## Ignore For Now

- Full public marketplace.
- Complex payments.
- AI reward generation.
- Large UI rewrite.
- Admin dashboard.

## Build Next Only After Saya Review

- Payment/subscription model.
- Private proof storage with signed URLs.
- Account deletion/privacy export.
- Group/household tenant model.
- Public launch branding and onboarding rewrite.
